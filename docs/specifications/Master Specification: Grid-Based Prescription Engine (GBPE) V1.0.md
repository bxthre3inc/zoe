# Master Specification: Grid-Based Prescription Engine V1.0

**Role**: Field Prescription Execution Layer | **Nodes**: DHU → PMT → SSN/ISN Fleet

The Grid-Based Prescription Engine (GBPE) is the software layer that translates a **field prescription map** generated by the Oracle engine into real-time, tiered hardware commands delivered across three heterogeneous control channels simultaneously.

## 1. Architecture Overview

```
[Oracle Engine @ DHU]
        │
        ├─── Uploads Field Prescription Map (FPM) → PMT Flash (via LoRa Long-Range)
        │
[PMT @ Pivot]
        │
        ├─── Tier 1: Computes angular speed commands from FPM
        ├─── Tier 2: Broadcasts SSN pressure setpoints (10x SSN on pivot span)
        └─── Tier 3: Unicasts/Multicasts ISN duty-cycle commands (125x ISN)
```

## 2. Field Prescription Map (FPM) Format

The FPM is a pre-compiled binary raster delivered to the PMT prior to each irrigation event.

| Field | Type | Description |
| :--- | :--- | :--- |
| `field_id` | UUID | Unique field identifier |
| `prescription_ts` | UNIX | Timestamp of Oracle computation |
| `grid_resolution` | ENUM | 1m × 1m pixel grid |
| `pixels[]` | ARRAY | Per-pixel records |
| `pixels[].lat_lng` | FLOAT[2] | Centroid of pixel |
| `pixels[].zone_id` | UINT8 | Multicast zone group (1–25) |
| `pixels[].water_mm` | FLOAT | Target application depth (mm) |
| `pixels[].tier` | ENUM | Speed / Section / Grid |

## 3. PMT Prescription Lookup (Real-Time Loop)

During active irrigation, the PMT runs a 1Hz lookup loop:

```
EVERY 1 SECOND:
  1. Read current pivot angle (RTK-GNSS arc position)
  2. Determine which pixel grid cells are within the active spray arc
  3. For each active pixel:
       a. Tier 1 → compute required angular velocity for target depth
       b. Tier 2 → compute flow/pressure setpoint for corresponding SSN span
       c. Tier 3 → compute PWM duty cycle for corresponding ISN nozzle(s)
  4. Broadcast Tier 2 SSN commands (CAN-Bus or LoRa)
  5. Multicast Tier 3 ISN zone commands (LoRa zone multicast)
  6. Log actuation telemetry to PMT flash
```

## 4. Zone Multicast Addressing (Tier 3)

To reduce downlink traffic, ISNs are grouped into **25 multicast zones** of ~5 ISNs each (across 125 units per field).

- Each zone covers approximately a 5m × 5m spray arc segment.
- The PMT issues **25 zone commands/sec** instead of **125 unicast commands/sec**.
- Individual ISN addresses override zone commands only for spot corrections.

## 5. Tier Escalation Logic

The GBPE selects the lowest-complexity tier sufficient to deliver the prescription:

| Pixel Delta (vs. Previous Pass) | Action |
| :--- | :--- |
| < 1mm deviation | No command (hold current state) |
| 1–5mm deviation | Tier 1: Speed adjustment only |
| 5–15mm deviation | Tier 2: SSN pressure adjustment |
| > 15mm deviation | Tier 3: ISN PWM command |

This prevents unnecessary radio traffic and hardware cycling for minor fluctuations.

## 6. Post-Irrigation Logging → Oracle Feedback

After each irrigation pass, the PMT uploads the actuation log to the DHU:

- Actual water applied per pixel vs. prescription
- SSN flow telemetry
- ISN actuation ack rates (% of commands confirmed)
- Grid cells where prescription could not be met (flagged for operator review)

The Oracle engine ingests this log to recalibrate the next prescription.

## 7. Prescription Format Conversion (Slice ↔ Grid)

The GBPE includes a bidirectional translation layer to ensure interoperability with legacy VRI systems (Valley, Lindsay, Reinke) that operate on **slice-based** (angular sector) prescriptions.

### 7.1 Legacy Slice → FarmSense Grid (Import)

For users migrating from a legacy system, the GBPE can import a slice prescription file and convert it:

1. **Ingest**: Parse the legacy sector file (typically CSV: `[angle_start, angle_end, application_mm]`).
2. **Rasterize**: Map each sector onto the corresponding 1m × 1m grid pixels using the PMT's field geometry registration.
3. **Refine**: Apply the **Soil Variability Map** to redistribute water within each sector based on actual soil texture — converting a single sector value into spatially-varied pixel values.

> [!NOTE]
> A single "135mm sector" in a legacy system may become pixels ranging from 110–160mm after SVM refinement, because the grid knows where the sandy pockets are.

### 7.2 FarmSense Grid → Legacy Slice (Export)

For pivot controllers that cannot accept grid prescriptions, the GBPE can down-convert:

1. **Aggregate**: Sum pixel water values within each angular sector.
2. **Average (Weighted)**: Weight by pixel area and soil-holding capacity.
3. **Output**: Produce a legacy-compatible CSV or ISOXML file.

> [!CAUTION]
> Exporting to slice format permanently loses the spatial precision of the grid prescription. The system will warn the operator of the expected yield precision loss estimate (typically 8-12% MAE increase).
